Sample DAX Measures

Reusable calculated measures applied across the star schema model

measures.dax DAX 
// Budget Variance % by Department
Budget Variance % =
VAR _Budget = SUM(Fact_Transactions[budget_amount])
VAR _Actual = SUM(Fact_Transactions[actual_amount])
RETURN
    DIVIDE(_Actual - _Budget, _Budget, 0)

// Utilization Rate (rolling 3-month)
Utilization Rate 3M =
CALCULATE(
    DIVIDE(
        SUM(Fact_Transactions[actual_amount]),
        SUM(Fact_Transactions[budget_amount])
    ),
    DATESINPERIOD(
        Dim_Date[date], MAX(Dim_Date[date]), -3, MONTH
    )
)

// Conditional Status Flag
Budget Status =
SWITCH(
    TRUE(),
    [Budget Variance %] > 0.03, "Over Budget",
    [Budget Variance %] > -0.02, "At Risk",
    "On Track"
)

PySpark Transformation Logic

Core transformation functions with validation and standardization

transform.py PySpark 
from pyspark.sql import SparkSession
from pyspark.sql import functions as F
from pyspark.sql.window import Window

spark = SparkSession.builder.appName("FinancialETL").getOrCreate()

# Standardize department names via lookup mapping
dept_map = {"engineering": "ENG", "eng.": "ENG",
            "operations": "OPS", "ops": "OPS"}

def clean_transactions(df):
    """Standardize, validate, and deduplicate transaction data."""

    # 1. Parse multiple date formats
    df = df.withColumn("date",
        F.coalesce(
            F.to_date(F.col("date"), "yyyy-MM-dd"),
            F.to_date(F.col("date"), "MM/dd/yyyy"),
            F.to_date(F.col("date"), "MMM dd yyyy")
        ))

    # 2. Map department names to standard IDs
    mapping_expr = F.create_map(
        [F.lit(x) for pair in dept_map.items() for x in pair])
    df = df.withColumn("dept_id",
        mapping_expr[F.lower(F.trim(F.col("dept")))])

    # 3. Handle nulls and negative values
    df = df.withColumn("amount",
        F.when(F.col("amount").isNull() | (F.col("amount") < 0), 0)
         .otherwise(F.col("amount")))

    # 4. Deduplicate on composite key
    w = Window.partitionBy("date", "dept_id", "amount") \
              .orderBy(F.col("date").desc())
    df = df.withColumn("rn", F.row_number().over(w)) \
           .filter(F.col("rn") == 1).drop("rn")

    return df

Why Interpretable ML Matters

Stakeholders don't trust black-box models. By extracting decision rules from XGBoost's tree structure and translating them into IF/THEN business logic, operations teams can validate the model against their domain expertise. This builds trust and drives adoption: the Streamlit app saw 85% weekly active usage within the first month of deployment.
